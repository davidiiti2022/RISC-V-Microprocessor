# RISC-V-Microprocessor
RISC-V 32-bit Pipelined Microprocessor (RV32I)
A 32-bit RISC-V pipelined microprocessor implemented in Verilog, supporting a subset of the RV32I instruction set. The processor follows a classical 5-stage pipeline architecture and resolves data hazards using forwarding logic to improve performance.

This project was developed to gain practical understanding of:

RISC-V instruction set architecture
5-stage pipeline design
Data hazard handling using forwarding
Control and datapath separation
RTL design and waveform-based verification

# Architecture Overview
The processor implements a standard 5-stage pipeline:

Instruction Fetch (IF)
Instruction Decode (ID)
Execute (EX)
Memory Access (MEM)
Write Back (WB)
Each stage is modular and implemented as separate Verilog modules.

Hazard Handling
Forwarding-Based Data Hazard Resolution
File:

Hazard_unit.v
The processor resolves RAW (Read After Write) data hazards using forwarding logic.

Forwarding paths implemented:

EX/MEM → EX stage
MEM/WB → EX stage
This allows dependent instructions to receive updated operand values without stalling the pipeline.

Note:
This design uses only forwarding logic for hazard resolution.
No pipeline stall mechanism or load-use stall logic is implemented.
